What is Sentry?

Sentry captures exceptions, groups them by root cause, and gives you stack traces, breadcrumbs, and context to debug faster.

Why with Encore?

Encore provides distributed tracing and structured logging. Sentry adds error alerting and reporting on top. The withSentry wrapper captures errors with endpoint name, request data, and breadcrumbs.

What to look for

After triggering an error, check your Sentry dashboard:

  • Stack trace
  • Endpoint tag
  • Request payload
  • Breadcrumb timeline

How it works

Sentry tracing is disabled (tracesSampleRate: 0) since Encore handles that. HTTP integrations are filtered out since Encore's Rust gateway handles inbound requests.
